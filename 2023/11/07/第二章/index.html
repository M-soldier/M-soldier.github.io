一切都是对象

一、程序的存储位置

  1. 寄存器
    最快的寄存器，位于处理器内部，不能直接控制，也不能在程序中感受到寄存器的存在。

  2. 堆栈
    位于RAM中，Java的一些数据存储于堆栈中（特别是对象引用，Java对象存储于堆中）。

  3. 堆
    一种通用的内存池，也位于RAM中，用于存放所有的Java对象，用堆进行存储分配和清理可能比用堆栈进行存储分配需要更多的时间（注意：Java不可以像C++一样在栈中创建对象）

  4. 常量存储
    常量值通常直接存放在程序代码内部，在嵌入式系统中，常量被存储在ROM中。（这种存储的一个例子是字符串池，详细内容查看jvm章节）

  5. 非RAM存储

    1. 流对象：在流对象中，对象转化成字节流，通常被发送给另一台及其；
    2. 持久化对象：在持久化对象中，对象被存放于磁盘上，Java提供了对轻量级持久化的支持。

二、关于Java中一些概念的说明

在Thinking In Java这本书中，作者提到了几个概念，分别是类、对象、对象引用。

  1. 类：类是某一类具有相同特征的对象集合，类名跟在关键词class后面；
  2. 对象：对象是某个类的具体示例；由new创建；
  3. 对象引用：对象引用可以看作是某一个对象的名称。

举个例子，“狗”是一个抽象的类，但是你家的哈士奇就是“狗”这个类中的一个实例，是一个对象，你家的狗名字叫“大黄”，那么“大黄”就是一个对象引用。只不过在程序当中，除了理解这几个概念的区别之外，还需要理解这几个概念之间的关系。

三、数据类型

Java数据类型分类如下：

$$
Java数据类型
\begin{cases}
基本数据类型
\begin{cases}
数值型
\begin{cases}
整数类型：byte[1], short[2], int[4], long[8]\
浮点数类型：float[4], double[8]\
\end{cases}\
字符型（存放单个字符，可以存放汉字）：char[2]\
布尔型：boolean[1]\
\end{cases}\
引用数据类型
\begin{cases}
类(class)\
接口(interface)\
数组([])\
\end{cases}\
\end{cases}
$$

其中基本数据类型直接存储值，存储在堆栈当中，同时，Java中的基本数据类型与其他大多数语言不同，其所占存储空间的大小与机器硬件架构无关，这也是Java程序可移植性高的一个原因。Java基本数据类型详细信息如下表所示：

基本类型 大小 最小值 最大值 包装器类型
boolean - - - Boolean
char 16-bit Unicode 0 Unicode $2^{16}-1$ Character
byte 8 bits $-128$ $+127$ Byte
short 16 bits $-2^{15}$ $+2^{15}-1$ Short
int 32 bits $-2^{31}$ $+2^{31}-1$ Integer
long 64 bits $-2^{63}$ $+2^{63}-1$ Long
float 32 bits IEEE754 IEEE754 Float
double 64 bits IEEE754 IEEE754 Double
void - - - Void

基本类型中具有的包装器类，可以在堆中创建一个非基本对象，用来表示对应的基本类型。

需要注意的是：

  • Java整型常量默认为int型，声明long型常量须后加”l”或”L”；
  • Java浮点型常量默认为double型，声明float型常量须后加”f”或”F”；
    • 浮点数有两种表示形式：
      • 十进制数形式：eg，5.12、512.0f、.512（必须有小数点）
      • 科学计数法形式：eg，5.12e2（等价于$5.1210^2$）、5.12E-2（等价于$5.1210^{-2}$）
    • 通常情况下应该使用double类型，更精确；
    • 浮点数使用陷阱：2.7与8.1/3比较大小。
  • 字符常量是用单引号扩起来的字符（包括转义字符），双引号为字符串；
    • 在Java中，char的本质是一个整数，在输出时，是unicode码对应的字符（ http://tool.chinaz.com/Tools/Unicode.aspx ）；
    • 可以直接给char赋一个整数，然后按照对应的unicode字符输出；
    • char可以进行运算，相当于一个整数。

高精度数字：

  1. BigInteger：支持任意精度的数字；
  2. BigDecimal：支持任意精度的定点数。
    注：上述两者都没有对应的基本类型，基本类型对应的操作必须以方法调用。

Java中的数组：

Java的主要目标之一是安全性，Java会确保数组被初始化，而且不能在它的范围之外被访问（这个是以每个数组上少量的内存开销和运行时的下标检查为代价的）。
当创建一个数组对象时，实际上就是创建了一个引用数组，并且每个引用自动被初始化为一个特定值，该值拥有自己的关键字null，表示还没有指向某个对象。在使用任何引用之前，必须为其指定一个对象，否则将会报错。

五、类的详解

字段和方法

一旦定义了类，就可以在类中设置两种类型的元素：字段（也叫数据成员）和方法（也叫成员函数）。如果字段是某个对象的引用，那么必须初始化该引用，以便使其与一个实际的对象相关联。
若类的某个成员是基本数据类型，即使没有初始化，Java也会赋给其一个默认值，如下表所示：

基本数据类型 默认值
boolean false
char (null) ‘\u0000’
byte (byte) 0
short (short) 0
int 0
long 0L
float 0.0f
double 0.0d

需要注意的是：只有类的某个成员是基本数据类型时才会自动初始化为默认值，若是在某个方法中变量，则不会初始化。
